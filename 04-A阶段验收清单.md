# A 阶段验收清单

更新时间：2026-05-14

## 1. 自动化验收

Server：

```bash
npm --prefix server run lint
npm --prefix server run test
npm --prefix server run test:e2e
npm --prefix server run build
```

Electron SDK：

```bash
npm --prefix sdk-electron run lint
npm --prefix sdk-electron run test
npm --prefix sdk-electron run build
```

Electron Demo：

```bash
npm --prefix demo-electron run lint
npm --prefix demo-electron run test
npm --prefix demo-electron run build
```

.NET SDK：

```bash
dotnet build sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

当前环境未安装 .NET SDK，.NET 构建需在安装 .NET 8 SDK 的机器或 CI 环境执行。

## 2. Electron Demo 人工验收

前置条件：

1. Server 已启动。
2. License UI 已构建或已启动。
3. seed 数据已写入数据库。
4. 使用 Demo License：`DEMO-AAAA-BBBB-CCCC`。

如服务端启用公共 API 签名，启动 Electron Demo 前设置签名密钥：

```bash
export LICENSE_API_BASE_URL=http://127.0.0.1:3000
export LICENSE_PRODUCT_CODE=demo_app
export LICENSE_REQUEST_SIGNING_SECRET=dev-public-api-signing-secret
```

如需要指定授权 UI 地址：

```bash
export LICENSE_UI_URL=http://127.0.0.1:5174
```

启动：

```bash
npm --prefix demo-electron run dev
```

验收步骤：

1. 首次打开 Demo，应显示未授权状态并弹出授权窗口。
2. 输入 `DEMO-AAAA-BBBB-CCCC`，激活应成功。
3. 主窗口授权状态应变为 `active`。
4. Feature Gate 应显示 `publish` 可用，`maxWindowCount` 有值。
5. 关闭 Demo 后重新打开，应可读取本地缓存并进入可用状态。
6. 点击“启动验证”，状态应保持可用。
7. 点击“开始心跳”，后台心跳日志应增加。
8. 后台封禁该 License 后，再点击“启动验证”，客户端应显示 banned 或不可用状态。
9. 后台版本策略设置最低版本高于 Demo 当前版本并开启强制升级后，再点击“启动验证”，客户端应显示强制升级相关错误。
10. 点击“解绑”并输入 License Key，后台设备状态应变为 removed。

验收记录建议保留：

- 操作日期和执行人。
- 操作系统版本。
- Server commit。
- Electron Demo 版本。
- 每一步结果截图。
- 失败步骤、错误码、日志片段和处理结论。

## 3. .NET SDK 联调验收

环境变量：

```bash
export ENTITLEMENT_API_BASE_URL=http://127.0.0.1:3000
export ENTITLEMENT_PRODUCT_CODE=demo_app
export ENTITLEMENT_LICENSE_KEY=DEMO-AAAA-BBBB-CCCC
export ENTITLEMENT_REQUEST_SIGNING_SECRET=dev-public-api-signing-secret
```

运行：

```bash
dotnet run --project sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

预期输出包含：

- `activate:`
- `verify:`
- `heartbeat:`
- `deactivate: OK`

后台应能看到对应激活日志、心跳日志和设备解绑状态变化。

## 4. 生产试运行巡检

启动后执行：

```bash
scripts/prod-healthcheck.sh
scripts/prod-backup.sh
scripts/prod-logs.sh server
```

检查项：

1. `docker compose ps` 中 postgres、server、admin、license-ui 均为 healthy。
2. `curl http://127.0.0.1:3000/health` 返回 `OK`。
3. 备份文件生成成功。
4. Admin 可登录。
5. 运行日志可按激活日志、心跳日志、审计日志切换查看。
6. SDK 接入页可打开，下载入口和接入说明可见。
7. `scripts/prod-logs.sh server` 可正常输出 Server 日志。
