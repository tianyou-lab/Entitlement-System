# 已完成开发文档

## 1. 项目概述

本项目是一个可复用于多个桌面软件的通用在线授权系统，覆盖 License 创建、在线激活、授权验证、心跳续租、设备绑定、远程封禁、功能位控制、强制升级、审计日志、风控、渠道、卡密、离线授权包、多 SDK 接入和生产部署基础设施。

当前代码已经完成原开发计划中的 P0、P1、P2 最小可运行版本，可进入桌面人工验收、生产密钥轮换和部署试运行阶段。

## 2. 当前工程结构

| 目录 | 说明 |
|---|---|
| `server/` | NestJS + Prisma 后端，提供公共授权 API、Admin API、风控与审计能力 |
| `admin/` | Vue 3 + Element Plus 管理后台 |
| `license-ui/` | 可嵌入的授权激活 UI |
| `sdk-electron/` | Electron/TypeScript SDK，含本地加密缓存和请求签名 |
| `demo-electron/` | Electron 接入示例 |
| `sdk-cpp/` | C++ SDK Demo |
| `sdk-dotnet/` | .NET SDK 最小客户端与 Demo |
| `docker-compose.prod.yml` | 生产部署编排 |

## 3. 已完成能力清单

### 3.1 P0 核心授权链路

- 产品管理：创建、列表、状态维护。
- 套餐管理：有效期、最大设备数、并发数、宽限时间、Feature Flags。
- License 管理：创建、列表、详情、状态修改、封禁。
- 公共授权 API：
  - `POST /api/v1/license/activate`
  - `POST /api/v1/license/verify`
  - `POST /api/v1/license/heartbeat`
  - `POST /api/v1/license/deactivate`
  - `GET /api/v1/license/info`
- 设备绑定：设备码、指纹、系统信息、应用版本、最后在线 IP、状态。
- Lease 机制：短期 lease token 签发、验证、续租、过期和撤销。
- 授权状态控制：active、inactive、expired、banned、suspended。
- 错误码统一：`OK`、`BAD_REQUEST`、`UNAUTHORIZED`、`LICENSE_BANNED`、`DEVICE_LIMIT_REACHED`、`LEASE_EXPIRED`、`FORCE_UPGRADE_REQUIRED` 等。
- 审计与业务日志：激活日志、心跳日志、管理员审计日志。

### 3.2 P1 上线增强能力

- Admin 登录和 JWT 鉴权。
- 生产 seed 必填 `ADMIN_PASSWORD`，开发默认管理员首次登录后要求修改密码。
- Admin UI：产品、套餐、License、设备、日志、版本策略管理。
- 设备管理：启用、移除、封禁。
- 日志查看：激活日志、心跳日志、审计日志。
- 版本策略：最低支持版本、最新版本、强制升级、下载地址、公告。
- 强制升级：客户端低于最低版本时返回 `FORCE_UPGRADE_REQUIRED`。
- 请求签名：公共 API 支持 HMAC 签名校验。
- nonce 防重放：公共 API 支持 nonce 去重和时间窗口校验。
- Electron SDK：activate / verify / heartbeat / deactivate，支持请求签名。
- Electron SDK 本地缓存：AES-256-GCM 加密。
- License UI：激活和授权状态展示基础能力。
- C++ SDK Demo：activate / verify / heartbeat / deactivate。
- 生产部署：Dockerfile、Docker Compose、环境变量示例、备份和回滚说明。

### 3.3 P2 扩展能力

- 风控面板：风险事件创建、列表、处理、统计摘要。
- Risk Service MVP 自动规则：
  - 单 License 24 小时激活次数超过阈值。
  - 单 License 多 IP 高频切换。
  - 同一设备指纹变化。
  - 单 License 高频解绑。
  - 异常版本号访问。
  - 心跳频率异常。
- 渠道/代理商：创建、列表、启停。
- 卡密：生成、发放、禁用、批次和渠道关联。
- 离线授权包：创建、撤销、恢复、签名、有效期校验。
- 离线授权导入接口：`POST /api/v1/offline-packages/import`。
- 自助设备解绑：客户端提交申请，后台通过/拒绝；通过后设备移除。
- 多租户基础模型：租户可关联产品、管理员、渠道、卡密、离线包、风险事件、保护器适配器。
- .NET SDK：activate / verify / heartbeat / deactivate / version policy 客户端源码，支持公共 API 请求签名。
- 保护器适配器：适配器登记、启停、配置和产品关联。
- GitHub Actions CI：覆盖 Server、Admin、License UI、Electron SDK/Demo、C++ SDK、.NET SDK 和 Docker 镜像构建。

## 8. 已知上线前注意事项

- 生产部署前必须轮换所有示例密钥。
- 生产部署后必须修改或删除 seed 默认管理员账号。
- 如启用 `PUBLIC_API_SIGNING_SECRETS`，客户端必须使用与 `productCode` / `appVersion` 精确匹配的签名密钥。
- 数据库备份、生产 `.env`、管理员凭据不得提交到仓库。
