# 通用授权系统-开发任务拆解 WBS

## 1. 文档目标

本文档用于将“通用在线授权系统”从产品与技术方案，进一步拆解为 **可执行开发任务**。  
目标是让后续工作可以直接进入：

- 排期
- 分工
- 开发
- 联调
- 测试
- 上线

本文档默认围绕 **MVP 到 V1** 范围拆解，不包含自研商业壳实现。

---

## 2. WBS 总览

整个项目拆为 10 个一级工作包：

1. 项目初始化与基础设施
2. 数据库与数据模型
3. 公共授权 API 服务
4. 后台管理系统
5. Electron SDK
6. C/C++ SDK
7. 通用授权 UI
8. 审计、风控与安全机制
9. 测试与联调
10. 部署与上线

## 7. MVP 最小闭环

如果只做最小闭环，应至少完成：

1. 产品/套餐/License 数据模型
2. 激活接口
3. 验证接口
4. 心跳接口
5. 后台创建 License
6. Electron SDK
7. 通用激活 UI
8. 基础日志

完成这 8 项，就已经具备首个可接入产品的能力。

---

## 8. 结论

本 WBS 的核心目的，是把“授权系统平台化”这件事从概念落到具体任务。  
建议从 **MVP 最小闭环** 开始，先确保：

- 一套服务端
- 一套后台
- 一套 Electron SDK
- 一套通用授权 UI

跑通后，再扩展到：

- C/C++ SDK
- 风控
- 渠道
- 离线授权
- 保护适配器
