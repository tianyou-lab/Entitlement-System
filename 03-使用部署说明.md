# 使用部署说明

## 1. 环境要求

基础环境：

- Node.js / npm
- PostgreSQL 16 或 Docker Compose
- GNU Make 和 g++（构建 C++ Demo）
- .NET 8 SDK（仅构建 `sdk-dotnet` 时需要）
- 桌面环境（仅 Electron GUI 人工验收时需要）

## 2. 本地开发启动

### 2.1 安装依赖

```bash
npm --prefix server install
npm --prefix admin install
npm --prefix license-ui install
npm --prefix sdk-electron install
npm --prefix demo-electron install
```

### 2.2 配置 Server 环境变量

开发环境至少需要：

```bash
export DATABASE_URL=postgresql://entitlement:entitlement@127.0.0.1:5432/entitlement?schema=public
export JWT_SECRET=dev-jwt-secret
export LEASE_SECRET=dev-lease-secret
# 可选：启用公共 API 请求签名
export PUBLIC_API_SIGNING_SECRET=dev-public-api-signing-secret
export LEASE_TTL_MINUTES=120
```

如未设置 `PUBLIC_API_SIGNING_SECRET`，公共授权 API 在开发环境允许不签名请求。

### 2.3 初始化数据库

```bash
npm --prefix server exec -- prisma generate --schema server/prisma/schema.prisma
npm --prefix server exec -- prisma db push --schema server/prisma/schema.prisma
npm --prefix server exec -- prisma db seed --schema server/prisma/schema.prisma
```

seed 默认数据：

| 类型 | 值 |
|---|---|
| Admin 用户名 | `admin` |
| Admin 密码 | 开发默认 `admin123456`，首次登录后必须修改；生产必须通过 `ADMIN_PASSWORD` 设置 |
| 产品编码 | `demo_app` |
| 套餐编码 | `basic` |
| Demo License | `DEMO-AAAA-BBBB-CCCC` |

生产环境必须修改默认管理员密码或删除默认管理员后重新创建。

### 2.4 启动服务

Server：

```bash
npm --prefix server run start:dev
```

Admin UI：

```bash
npm --prefix admin run dev
```

License UI：

```bash
npm --prefix license-ui run dev
```

Electron Demo：

```bash
npm --prefix demo-electron run dev
```

## 4. 公共授权 API

统一响应格式：

```json
{
  "code": "OK",
  "message": "success",
  "data": {}
}
```

### 4.1 激活

```http
POST /api/v1/license/activate
```

请求示例：

```json
{
  "productCode": "demo_app",
  "licenseKey": "DEMO-AAAA-BBBB-CCCC",
  "device": {
    "deviceCode": "device-001",
    "fingerprintHash": "fingerprint-hash",
    "deviceName": "Office PC",
    "osType": "linux",
    "osVersion": "6.12",
    "appVersion": "1.0.0",
    "hardwareSummary": {
      "cpu": "test"
    }
  }
}
```

### 4.2 验证

```http
POST /api/v1/license/verify
```

```json
{
  "productCode": "demo_app",
  "leaseToken": "lease-token",
  "deviceCode": "device-001",
  "appVersion": "1.0.0"
}
```

### 4.3 心跳续租

```http
POST /api/v1/license/heartbeat
```

```json
{
  "productCode": "demo_app",
  "leaseToken": "lease-token",
  "deviceCode": "device-001",
  "appVersion": "1.0.1",
  "runtime": {
    "sessionId": "session-001"
  }
}
```

### 4.4 解绑

```http
POST /api/v1/license/deactivate
```

```json
{
  "productCode": "demo_app",
  "licenseKey": "DEMO-AAAA-BBBB-CCCC",
  "deviceCode": "device-001"
}
```

### 4.5 版本策略

```http
GET /api/v1/version/policy?productCode=demo_app
```

### 4.6 离线授权包导入

```http
POST /api/v1/offline-packages/import
```

```json
{
  "packageCode": "OFFLINE-XXXX-YYYY",
  "signature": "signature",
  "payload": {}
}
```

### 4.7 自助解绑申请

```http
POST /api/v1/device-unbind-requests
```

```json
{
  "licenseKey": "DEMO-AAAA-BBBB-CCCC",
  "deviceCode": "device-001",
  "reason": "换电脑"
}
```

## 5. 请求签名

如果设置了 `PUBLIC_API_SIGNING_SECRET`，公共 API 需要携带：

- `x-entitlement-timestamp`
- `x-entitlement-nonce`
- `x-entitlement-signature`

签名原文由服务端按以下字段生成：

```text
METHOD\nURL\nTIMESTAMP\nNONCE\nSTABLE_JSON_BODY
```

签名算法：HMAC-SHA256，输出 base64url。

时间戳窗口：5 分钟。nonce 在窗口内不可重复使用。

## 6. Admin 使用说明

1. 启动 Server。
2. 启动 Admin UI。
3. 使用 seed 管理员登录：
   - 用户名：`admin`
   - 密码：开发默认 `admin123456`，生产使用 `.env.production` 中的 `ADMIN_PASSWORD`
4. 如果系统提示修改默认密码，请先设置至少 12 位的新密码。
5. 创建或使用 seed 产品 `demo_app`。
5. 创建套餐。
6. 创建 License。
7. 在客户端或 Demo 中激活 License。
8. 在后台查看设备、日志、风险事件。
9. 可按需封禁 License、移除设备、开启强制升级、生成卡密或离线授权包。

## 7. SDK 使用说明

### 7.1 Electron SDK

配置项包括：

- `apiBaseUrl`
- `productCode`
- `requestSigningSecret`（可选）

主要能力：

- activate
- verify
- heartbeat
- deactivate
- 本地授权缓存加密
- 请求签名

### 7.2 C++ SDK Demo

构建：

```bash
make -C sdk-cpp clean all
```

运行前确保 Server 已启动，并准备有效 License。

### 7.3 .NET SDK Demo

构建：

```bash
dotnet build sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

运行：

```bash
export ENTITLEMENT_API_BASE_URL=http://127.0.0.1:3000
export ENTITLEMENT_PRODUCT_CODE=demo_app
export ENTITLEMENT_LICENSE_KEY=DEMO-AAAA-BBBB-CCCC
dotnet run --project sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

如服务端启用了公共 API 请求签名，同时设置：

```bash
export ENTITLEMENT_REQUEST_SIGNING_SECRET=dev-public-api-signing-secret
```

## 8. 生产部署

### 8.1 准备环境变量

复制示例文件：

```bash
cp .env.production.example .env.production
```

填写生产随机密钥：

```env
POSTGRES_PASSWORD=replace-with-long-random-password
ADMIN_PASSWORD=replace-with-long-random-admin-password
JWT_SECRET=replace-with-long-random-jwt-secret
LEASE_SECRET=replace-with-long-random-lease-secret
PUBLIC_API_SIGNING_SECRET=replace-with-long-random-public-api-signing-secret
LEASE_TTL_MINUTES=120
```

### 8.2 启动生产服务

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

### 8.3 初始化数据库

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec server npx prisma db push
docker compose --env-file .env.production -f docker-compose.prod.yml exec server npx prisma db seed
```

### 8.4 日常检查

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
docker compose --env-file .env.production -f docker-compose.prod.yml logs --tail=200 server
```

### 8.5 数据备份

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec postgres pg_dump -U entitlement entitlement > entitlement-backup.sql
```

### 8.6 回滚

1. 保留旧镜像或旧代码 tag。
2. 停止服务。
3. 恢复旧镜像或旧代码并重新启动。
4. 如数据库 schema 已变更，先确认是否需要恢复备份。

## 9. 生产安全要求

- 不要提交 `.env.production`。
- 不要提交数据库备份。
- 不要提交管理员凭据。
- 首次部署后必须修改默认管理员密码。
- 生产环境必须使用高强度随机 `JWT_SECRET`、`LEASE_SECRET`、`PUBLIC_API_SIGNING_SECRET`。
- 开启 HTTPS。
- 定期备份 PostgreSQL。
- 定期轮换密钥。
