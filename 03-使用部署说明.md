# 使用部署说明

## 1. 环境要求

基础环境：

- Node.js / npm
- PostgreSQL 16 或 Docker Compose
- GNU Make 和 g++（构建 C++ Demo）
- .NET 8 SDK（仅构建 `sdk-dotnet` 时需要）
- 桌面环境（仅 Electron GUI 人工验收时需要）

## 2. 本地开发启动

### 2.1 安装依赖

```bash
npm --prefix server install
npm --prefix admin install
npm --prefix license-ui install
npm --prefix sdk-electron install
npm --prefix demo-electron install
```

### 2.2 配置 Server 环境变量

开发环境至少需要：

```bash
export DATABASE_URL=postgresql://entitlement:entitlement@127.0.0.1:5432/entitlement?schema=public
export JWT_SECRET=dev-jwt-secret
export LEASE_SECRET=dev-lease-secret
# 可选：启用公共 API 请求签名
export PUBLIC_API_SIGNING_SECRET=dev-public-api-signing-secret
export LEASE_TTL_MINUTES=120
export PUBLIC_API_RATE_LIMIT_MAX=120
export PUBLIC_API_RATE_LIMIT_WINDOW_MS=60000
```

如未设置 `PUBLIC_API_SIGNING_SECRET`，公共授权 API 在开发环境允许不签名请求。

### 2.3 初始化数据库

```bash
npm --prefix server exec -- prisma generate --schema server/prisma/schema.prisma
npm --prefix server exec -- prisma migrate dev --schema server/prisma/schema.prisma
npm --prefix server exec -- prisma db seed --schema server/prisma/schema.prisma
```

seed 默认数据：

| 类型 | 值 |
|---|---|
| Admin 用户名 | `admin` |
| Admin 密码 | 开发默认 `admin123456`，首次登录后必须修改；生产必须通过 `ADMIN_PASSWORD` 设置 |
| 产品编码 | `demo_app` |
| 套餐编码 | `basic` |
| Demo License | `DEMO-AAAA-BBBB-CCCC` |

生产环境必须修改默认管理员密码或删除默认管理员后重新创建。

### 2.4 启动服务

Server：

```bash
npm --prefix server run start:dev
```

Admin UI：

```bash
npm --prefix admin run dev
```

License UI：

```bash
npm --prefix license-ui run dev
```

Electron Demo：

```bash
export LICENSE_API_BASE_URL=http://127.0.0.1:3000
export LICENSE_PRODUCT_CODE=demo_app
export LICENSE_REQUEST_SIGNING_SECRET=dev-public-api-signing-secret
npm --prefix demo-electron run dev
```

## 4. 公共授权 API

## 5. 请求签名

如果设置了 `PUBLIC_API_SIGNING_SECRET`，公共 API 需要携带：

- `x-entitlement-timestamp`
- `x-entitlement-nonce`
- `x-entitlement-signature`

签名原文由服务端按以下字段生成：

```text
METHOD\nURL\nTIMESTAMP\nNONCE\nSTABLE_JSON_BODY
```

签名算法：HMAC-SHA256，输出 base64url。

时间戳窗口：5 分钟。nonce 在窗口内不可重复使用。

## 6. 公共 API 限流

公共客户端 API 默认启用轻量内存限流，作用于：

- `/api/v1/license/*`
- `/api/v1/version/*`
- `/api/v1/offline-packages/*`
- `/api/v1/device-unbind-requests`

默认策略为每个客户端 IP、每个接口路径每 60 秒最多 120 次请求。可通过环境变量调整：

```bash
export PUBLIC_API_RATE_LIMIT_MAX=120
export PUBLIC_API_RATE_LIMIT_WINDOW_MS=60000
```

超过限制时返回：

```json
{
  "code": "RATE_LIMITED",
  "message": "too many requests",
  "data": null
}
```

## 7. Admin 使用说明

1. 启动 Server。
2. 启动 Admin UI。
3. 使用 seed 管理员登录：
   - 用户名：`admin`
   - 密码：开发默认 `admin123456`，生产使用 `.env.production` 中的 `ADMIN_PASSWORD`
4. 如果系统提示修改默认密码，请先设置至少 12 位的新密码。
5. 创建或使用 seed 产品 `demo_app`。
5. 创建套餐。
6. 创建 License。
7. 在客户端或 Demo 中激活 License。
8. 在后台查看设备、日志、风险事件。
9. 可按需封禁 License、移除设备、开启强制升级、生成卡密或离线授权包。

## 9. 生产部署

### 9.1 准备环境变量

复制示例文件：

```bash
cp .env.production.example .env.production
```

填写生产随机密钥：

```env
POSTGRES_PASSWORD=replace-with-long-random-password
ADMIN_PASSWORD=replace-with-long-random-admin-password
JWT_SECRET=replace-with-long-random-jwt-secret
LEASE_SECRET=replace-with-long-random-lease-secret
PUBLIC_API_SIGNING_SECRET=replace-with-long-random-public-api-signing-secret
LEASE_TTL_MINUTES=120
PUBLIC_API_RATE_LIMIT_MAX=120
PUBLIC_API_RATE_LIMIT_WINDOW_MS=60000
```

### 9.2 启动生产服务

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

### 9.3 初始化数据库

```bash
scripts/prod-migrate.sh
docker compose --env-file .env.production -f docker-compose.prod.yml exec server npx prisma db seed
```

生产环境使用 Prisma migration，迁移文件位于 `server/prisma/migrations/`。后续 schema 变更流程：

```bash
npm --prefix server exec -- prisma migrate dev --schema server/prisma/schema.prisma
npm --prefix server exec -- prisma validate --schema server/prisma/schema.prisma
```

提交前审核新生成的 `server/prisma/migrations/*/migration.sql`，生产执行前先备份数据库。生产环境不要直接执行 `db push`。

### 9.4 生产辅助脚本

项目根目录提供以下脚本：

```bash
scripts/prod-migrate.sh
scripts/prod-backup.sh
scripts/prod-healthcheck.sh
scripts/prod-logs.sh
```

可通过环境变量覆盖默认配置：

```bash
ENV_FILE=.env.production COMPOSE_FILE=docker-compose.prod.yml scripts/prod-migrate.sh
BACKUP_DIR=backups scripts/prod-backup.sh
ADMIN_URL=http://127.0.0.1:8080 SERVER_URL=http://127.0.0.1:3000 LICENSE_UI_URL=http://127.0.0.1:8081 scripts/prod-healthcheck.sh
TAIL=200 scripts/prod-logs.sh server
FOLLOW=1 scripts/prod-logs.sh all
```

### 9.5 日常检查

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
scripts/prod-logs.sh server
curl http://127.0.0.1:3000/health
```

### 9.6 数据备份

```bash
scripts/prod-backup.sh
```

默认备份输出到 `backups/entitlement-YYYYmmdd-HHMMSS.sql`。可通过 `BACKUP_DIR` 覆盖目录。

### 9.7 备份恢复演练

恢复演练建议在测试环境执行，不要直接覆盖生产库。

1. 准备一份最近备份文件。
2. 启动一套独立测试环境，使用新的 PostgreSQL volume。
3. 执行恢复：

```bash
BACKUP_FILE=backups/entitlement-YYYYmmdd-HHMMSS.sql
docker compose --env-file .env.production -f docker-compose.prod.yml exec -T postgres psql -U entitlement -d entitlement < "$BACKUP_FILE"
```

4. 执行迁移和 seed 校验：

```bash
scripts/prod-migrate.sh
scripts/prod-healthcheck.sh
```

5. 登录 Admin，抽查产品、授权卡密、设备、运行日志、离线包和风险事件。
6. 使用 Demo License 完成一次激活、验证、心跳。
7. 记录恢复耗时、备份文件大小、异常项和处理结论。

### 9.8 回滚

1. 保留旧镜像或旧代码 tag。
2. 停止服务。
3. 恢复旧镜像或旧代码并重新启动。
4. 如数据库 schema 已变更，先确认是否需要恢复备份。

详细迁移审核和回滚流程见 `05-数据库迁移与回滚规范.md`。

## 10. 生产安全要求

- 不要提交 `.env.production`。
- 不要提交数据库备份。
- 不要提交管理员凭据。
- 首次部署后必须修改默认管理员密码。
- 生产环境必须使用高强度随机 `JWT_SECRET`、`LEASE_SECRET`、`PUBLIC_API_SIGNING_SECRET`。
- 开启 HTTPS。
- 定期备份 PostgreSQL。
- 定期轮换密钥。
