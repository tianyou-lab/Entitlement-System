# 待开发计划文档

## 1. 当前结论

原计划中的 P0、P1、P2 功能已经完成最小可运行版本。生产默认管理员密码处理、首次登录强制改密、GitHub Actions CI、.NET SDK 请求签名、管理员后台 SDK 下载入口和接入文档也已完成。后续待开发内容不再是核心授权链路补缺，而是围绕真实环境验收、生产化、商业化、可运营性和更高安全等级继续增强。

## 2. 必做验收项

### 2.1 桌面 GUI 人工验收

当前环境为 headless，无法实际打开 Electron GUI。需要在桌面环境完成：

1. 启动 Server。
2. 启动或打包 Electron Demo。
3. 输入 seed License：`DEMO-AAAA-BBBB-CCCC`。
4. 验证激活成功、重启后本地缓存可用。
5. 验证 heartbeat 后 lease token 刷新。
6. 后台封禁 License 后，客户端再次验证应失效。
7. 后台开启强制升级后，低版本客户端应提示升级。

### 2.2 .NET SDK 构建验收

当前机器未安装 .NET SDK。需要在安装 .NET 8 SDK 的环境执行：

```bash
dotnet build sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

如需要真实联调，再设置：

```bash
export ENTITLEMENT_API_BASE_URL=http://127.0.0.1:3000
export ENTITLEMENT_PRODUCT_CODE=demo_app
export ENTITLEMENT_LICENSE_KEY=DEMO-AAAA-BBBB-CCCC
dotnet run --project sdk-dotnet/demo/Entitlement.Sdk.Demo.csproj
```

## 3. 生产化增强计划

### 3.1 数据库迁移体系

当前项目已有 Prisma schema 和初始化 SQL。后续建议改为严格 migration 流程：

- 使用 Prisma migration 管理每次 schema 变更。
- 为生产环境建立迁移前备份流程。
- 建立 migration 回滚或修复预案。
- CI 中校验 schema 与 migration 一致性。

### 3.2 CI/CD

已新增 GitHub Actions 流水线，覆盖：

1. Server lint / test / e2e / build。
2. Admin lint / test / build。
3. License UI lint / test / build。
4. Electron SDK lint / test / build。
5. Electron Demo lint / test / build。
6. C++ SDK build。
7. .NET SDK build。
8. Docker image build。

后续如进入生产发布阶段，可继续补充自动部署、制品归档和环境审批。

### 3.3 生产监控

建议补充：

- API 请求量、失败率、延迟监控。
- 激活失败率、设备超限、License banned、force upgrade 命中率。
- 风险事件数量、未处理高危风险数量。
- PostgreSQL 连接数、存储、慢查询。
- 服务健康检查和告警。

### 3.4 安全加固

已完成：

- 生产环境 seed 必须提供 `ADMIN_PASSWORD`。
- 管理员首次登录强制修改默认密码。
- 公共 API 支持 HMAC 请求签名和 nonce 防重放。

建议继续增强：

- Admin 多角色权限细分。
- Admin 操作二次确认。
- 公共 API 限流。
- IP 黑名单 / 白名单。
- HMAC 密钥轮换机制。
- 更严格的设备指纹策略。
- 管理后台 HTTPS 与安全响应头。
- 生产日志脱敏。

## 4. 商业化增强计划

### 4.1 支付与订单

当前系统管理 License 和卡密，但没有支付闭环。后续可增加：

- 订单模型。
- 支付回调。
- 自动发码。
- 退款后自动禁用 License。
- 发票、对账和销售统计。

### 4.2 订阅模式

建议增加：

- 订阅周期。
- 自动续费状态。
- 宽限期。
- 续费失败后的授权降级或暂停。
- 套餐升级和降级。

### 4.3 渠道结算

当前已有渠道/代理商基础模型，后续可增强：

- 渠道价格策略。
- 渠道专属卡密批次。
- 渠道销售统计。
- 渠道佣金结算。
- 渠道管理员权限隔离。

## 5. 多租户增强计划

当前多租户为基础模型级实现。若要 SaaS 化，需要继续补：

- 所有 Admin 查询按 tenant 隔离。
- 管理员登录后绑定 tenant 上下文。
- 超级管理员跨租户管理能力。
- 租户套餐、配额和到期时间。
- 租户级审计日志。
- 租户级密钥和品牌配置。

## 6. 离线授权增强计划

当前已支持离线授权包创建、签名和导入校验。后续可增强：

- 离线包导出文件格式。
- 客户端离线包导入 UI。
- 离线包使用次数限制。
- 离线包绑定设备指纹。
- 离线包撤销列表同步。
- 离线包过期前提醒。

## 7. 保护器适配增强计划

当前已有保护器适配器登记模型。后续可扩展：

- 保护器命令行接口标准。
- 构建前授权注入。
- 加壳后签名校验。
- Loader 启动前在线授权验证。
